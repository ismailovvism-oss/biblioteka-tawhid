Хронология для самопогружения в контекст. Детали формата — `SPEC.md`, рабочий
контекст — `CLAUDE.md`. Новое сверху.

- `#btn-prev`/`#btn-next` крупнее и контрастнее (1.6rem, жирные, цвет акцента).

## 2026-06-19 — валидатор не мешает читателю (sw `v25`)
- Убрана проверка «глава целиком только в один язык» — для гибрида ru-only глава
  (вступление/проза) это норма, не предупреждение (`parser.js`).
- Значок ⚠ и панель валидатора — **только в режиме debug** (инструмент автора, не
  для читателя): `btn.hidden = !debug || !warnings`. Обычный читатель ⚠ не видит.

## 2026-06-19 — фиксы UX: главы без пары, страница, панель валидатора (sw `v24`)

### `app.js`
- **Глава без файла на одном языке больше не падает.** `loadChapterData`/`searchChapter`
  ловят 404 на язык-файл (гибрид: глава только на одном языке) → пустая строка, глава
  рендерится односторонней. Раньше — «Не удалось загрузить главу … HTTP 404».
- **Индикатор страницы:** нет маркера `<!-- pN -->` (статьи/главы без страниц) → не
  показываем «стр. —», только процент.
- **Панель валидатора** не рисует зелёную «ошибок не найдено» (зря занимала место):
  показывается только в debug И только когда есть реальные предупреждения.

## 2026-06-19 — статус доверия книги: бейдж + фасет (sw `v20`)

### `app.js` / `style.css` — поле `review`
- `review` (одно значение): `approved` (✓ Одобрено), `caution` (⚠ Требует проверки),
  `draft` (🚧 В работе). `reviewBadge()` рисует **цветной бейдж** в углу обложки —
  на полке (иконка) и на карточке книги (иконка + подпись). `REVIEW`-словарь.
- Фасет **«Статус»** в `FACETS` (первым) — фильтр по тому же полю. Книга без `review`
  — без бейджа. Уровень книги (доверие для читателя), не путать со статусом раздела.

## 2026-06-19 — фасеты «Мазхаб» и «Жанр» (sw `v19`)

### `app.js` — два новых фасета каталога
- `FACETS` += `{madhhab:'Мазхаб'}` и `{genre:'Жанр'}` (поля `madhhab[]`/`genre[]`
  в записи `index.json`). Логика прежняя: показываются при ≥2 значениях, внутри
  группы ИЛИ, между группами И. Мазхаб (ханафи/малики/шафии/ханбали/захири/
  сравнительный) и жанр (рудуд/шарх/матн/фатавы) — ортогональны дереву разделов.
- Проставляются из формы публикации вычитки; дерево разделов (`category`) —
  иерархия любой глубины, см. `CLAUDE.md`.

## 2026-06-18 — сворачиваемые callout’ы (sw `v18`)

### `parser.js` / `style.css` — фолд callout’ов как в Obsidian
- `renderBlocks`: `> [!тип]+` → раскрытый сворачиваемый callout, `> [!тип]-` →
  свёрнутый; без знака — статичный `<div>` как раньше. Регэксп `[!тип]([+-]?)`.
  Сворачиваемые рендерятся как `<details class="callout-foldable">` + `<summary>`.
- CSS: маркер-стрелка у `<summary>`, тело свёрнутого скрыто явным правилом
  `details.callout-foldable:not([open]) > .callout-body{display:none}` (не полагаемся
  на UA-стиль `<details>`). Тот же словарь и рендер — в вычитке (редактор).

## 2026-06-16 — общий словарь с вычиткой, заметки, недопереведённые секторы

Вычитка (`translator-panel`) стала редактором этого формата «на месте»; читалка
доработана под общий контракт.

### `parser.js` — рендер форматирования (sw `v17`)
- `inlineMd`: подчёркивание `<u>…</u>`.
- `renderGroup`→`renderBlocks`: цитата `> ` → `<blockquote>`; списки `- `/`1. ` →
  `<ul>/<ol>`; callout `> [!тип] заголовок` → `.callout.callout-<тип>` (тип
  произвольный, дефолтная подпись из `CALLOUT_LABELS`, неизвестный тип — базовый стиль).
- `parseFile`: регионы `<!-- note --> … <!-- /note -->` пропускаются (личные
  редакторские заметки автора не публикуются). Парсер остаётся DOM-free.

### `app.js` / `style.css` — недопереведённые секторы
- `renderChapter`: текст-пара с одним показанным языком → `.solo` (во всю ширину);
  оригинал без перевода (в двуязычной книге) → `.awaiting` (пунктирная акцентная
  грань + значок «⟳», слово — в `title`, видно только при параллельном чтении).
- CSS форматирования (blockquote/списки/callout/`<u>`) — тема-зависимый.

### Зачем
Гибридные книги (моноязычный «хребет» + вставные параллельные цитаты) и публикация
недопереведённых книг: сектор только в оригинале = «перевод готовится», сектор только
в переводе = основной текст во всю ширину, сектор в обоих с одним id = параллель.
