# Параллельная читалка Markdown-книг

Статическое веб-приложение для параллельного чтения книги в двух языках
(оригинал + перевод), выровненных по смысловым секторам. Vanilla JS, без
фреймворков и сборщиков. Подробная спецификация — в [SPEC.md](SPEC.md).

## Запуск

Нужен любой статический сервер (из-за `fetch` контента файлы нельзя открывать
через `file://`):

```bash
python3 -m http.server 8000
# или: npx serve .
```

Открыть `http://localhost:8000`. Стартовый экран — **библиотека** (авторский
список книг из `books/index.json`); тап по книге открывает её. Прямая ссылка на
книгу — параметр `?book=<bookId>`; кнопка «📚» в шапке возвращает к списку.

## Структура

```
index.html        разметка-каркас
style.css         стили (мобайл-фёрст; раскладки — CSS-состояния)
parser.js         ядро: парсер контракта, модель пар, валидатор (без DOM)
app.js            приложение: рендер потока, режимы, навигация, настройки
books/index.json  авторский список книг (что показывать в библиотеке)
books/<bookId>/   контент (грузится в рантайме, в код не бандлится)
    ar/NN.md      оригинал по главам
    ru/NN.md      перевод, те же имена файлов
    img/pNNN.jpg  сканы страниц (опционально)
    book.json     манифест (названия, порядок глав, rtl-языки)
books/_sample/    тестовая мини-книга для разработки
tools/validate.js CLI-валидатор контракта (Node.js)
```

Формат контента (якоря секторов `<!-- s007 -->`, сноски `<!-- fn073 -->` / `[^73]`,
страницы `<!-- p152 -->`, группы `s050a/b`) описан в SPEC.md, раздел 3.

## Возможности v1

- Единый поток пар «оригинал + перевод», выравнивание по id секторов.
- Видимость языков: оба / только оригинал / только перевод (кнопка в шапке).
- Раскладка пары: вертикальная или две колонки — автоматически по ориентации
  экрана, либо вручную в настройках.
- RTL/LTR по языку контента.
- Сноски двумя механиками (настройка): разворот на месте или скачок-возврат.
- Оглавление, переход по номеру страницы (сквозная нумерация по тому),
  показ скана страницы (если у книги есть `img/`).
- Валидатор контракта: рассинхрон id, битые/висячие сноски, сектора без
  страницы. При ошибках в шапке появляется «⚠ N»; полный список — в панели
  за настройкой «режим проверки».
- Настройки (тема, раскладка, режим сносок, видимость) — в `localStorage`.

## Валидатор из консоли

Для проверки разметки тома без браузера (требуется Node.js):

```bash
node tools/validate.js books/_sample
```

Выводит по главам число секторов/сносок/страниц и все ошибки контракта;
код выхода 1 при ошибках — удобно для CI.

## Список книг (`books/index.json`)

Библиотека формируется из авторского списка — это **единственный файл, который
правишь, чтобы добавить/убрать/переупорядочить книги**. Сам контент книг при
этом не трогается. Формат:

```json
{
  "title": "Библиотека",
  "books": [
    { "id": "_sample", "base": "books/_sample/",
      "title": { "ar": "كتاب تجريبي", "ru": "Тестовая мини-книга" } }
  ]
}
```

- `id` — идентификатор для ссылки `?book=<id>` и для запоминания последней главы.
- `base` — префикс путей книги (читалка грузит `base + "book.json"`,
  `base + "ar/NN.md"` и т.д.). Может быть:
  - **локальным путём** — `"books/<id>/"` (контент в этом репозитории);
  - **внешним URL** — книга остаётся в своём репозитории и грузится в рантайме,
    в читалку не копируется. Например, GitHub-репозиторий через raw:
    `"https://raw.githubusercontent.com/<owner>/<repo>/<branch>/"`
    (raw отдаёт `access-control-allow-origin: *`, поэтому CORS из браузера ок).
- `title` — как показать в библиотеке (берётся `ru`, ниже мелким — `ar`).

Внешняя книга обязана лежать по `base` в формате Контракта: `book.json`, папки
языков с якорями секторов и т.д. (SPEC.md, раздел 3). Иначе главы откроются
пустыми/с предупреждениями валидатора.

## Добавление книги

1. Подготовить контент по Контракту (SPEC.md, раздел 3): `book.json`, языковые
   папки, якоря секторов/сносок/страниц. Образец — `books/_sample/`.
2. Разместить его — локально в `books/<bookId>/` либо в отдельном публичном
   репозитории (тогда `base` будет внешним URL).
3. Прогнать валидатор: `node tools/validate.js books/<bookId>` (для локальных).
4. Добавить запись в `books/index.json`.
5. Открыть `?book=<bookId>` или выбрать книгу в библиотеке.
